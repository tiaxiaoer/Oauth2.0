JWT客户端认证流程 - 方法执行顺序

下图展示了Keycloak中JWT客户端认证流程中四个核心方法的执行顺序：readJws、validateClient、validateTokenAudience和validateToken。这些方法在JWTClientValidator类中定义，并由JWTClientAuthenticator和JWTClientSecretAuthenticator调用。

graph TD A[开始认证流程] B[clientAssertion参数验证] C[readJws 读取并解析JWS] D{readJws成功?} E[validateClient 验证客户端身份] F{validateClient成功?} G[验证签名算法] H{签名算法验证成功?} I[验证JWT签名] J{签名验证成功?} K[validateTokenAudience 验证令牌受众] L{受众验证成功?} M[validateToken 验证令牌有效性] N{令牌验证成功?} O[验证令牌重用] P{令牌重用验证成功?} Q[认证成功] R[认证失败] A --> B B --> C C --> D D -- 是 --> E D -- 否 --> R E --> F F -- 是 --> G F -- 否 --> R G --> H H -- 是 --> I H -- 否 --> R I --> J J -- 是 --> K J -- 否 --> R K --> L L -- 是 --> M L -- 否 --> R M --> N N -- 是 --> O N -- 否 --> R O --> P P -- 是 --> Q P -- 否 --> R

图例说明

readJws：读取客户端提供的JWT断言，并解析为JsonWebToken对象
validateClient：验证客户端身份，检查client_id是否有效且已启用
validateTokenAudience：验证令牌的受众声明，确保与当前Keycloak服务器匹配
validateToken：验证令牌的基本有效性，包括检查令牌是否过期、是否在未来签发等
